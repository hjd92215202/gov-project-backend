# 后端测试说明

## 职责
- 说明后端测试分层、执行命令和维护约定。

## 为什么存在
- 保护权限聚合、项目状态机、审批流和接口载荷组装。
- 让 `mvn test` 能在不依赖本地真实数据库的前提下稳定执行。

## 关键输入输出
- 单元测试目录：`src/test/java`
- 执行命令：
  - `C:\\apache-maven-3.9.11\\bin\\mvn.cmd test`
  - `C:\\apache-maven-3.9.11\\bin\\mvn.cmd -q -DskipTests compile`

## 关联链路
- 登录与当前用户信息
- 用户访问上下文
- 项目新增 / 更新 / 提交
- 审批待办与审批流转

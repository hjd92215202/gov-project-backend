# 后端开发前 / 提测前检查清单

## 开发前（5分钟）
1. 确认 JDK 版本：`java -version` 应为 `1.8`（团队当前标准：`jdk1.8.0_112`）。
2. 确认 Maven 可用：`mvn -v`（版本建议 `3.9.11`）。
3. 检查配置文件：`application.yml`、环境变量（尤其 `GOV_SM4_KEY`、`GOV_PASSWORD_PEPPER`、`LOG_PATH`）。
4. 检查编码规范：IDE 文件编码为 `UTF-8 without BOM`，遵守 `.editorconfig`。
5. 本地数据库联通与账号确认（MariaDB/openGauss）。

## 开发中（提交前）
1. 自测关键链路：登录、`/system/me`、项目分页、项目提交审批、审批通过/驳回、地图接口。
2. 确认权限边界：管理员、部门负责人、普通用户各自操作是否符合预期。
3. 检查日志安全：不在日志里输出密码、token、明文敏感字段。
4. 检查异常提示：返回 `R(code,msg,data)` 结构且中文提示清晰。

## 提测前（必跑）
1. 单元/接口测试：`mvn test`
2. 若改动数据库结构：执行 patch 并做 `EXPLAIN` 验证索引命中。
3. 手工回归：
   - 登录与权限
   - 用户分页与角色分配
   - 项目新增/编辑/删除/提交审批
   - 审批待办/已办与通过/驳回
   - 地图加载
4. 检查日志文件输出：
   - `./logs/app.log`
   - `./logs/error.log`
   - `./logs/audit.log`

## 发布前（建议）
1. 环境变量改为正式值（尤其国密 key 与 pepper）。
2. 关闭非必要调试日志级别。
3. 备份数据库与配置快照。
